# LimeStudio スクリプティングガイド：サンドボックス

LimeStudio において、スクリプティングは **「シグナルグラフを生成するため」** に使用され、サンプルを直接処理するためではありません。このユニークなアプローチにより、カスタムロジックを組み込みノードと同じ速度で実行できることが保証されます。

---

## 1. コア・コンセプト

LimeStudio の `Script` ノードは、**コンパイル時のジェネレーター** です。
あなたがスクリプトを書くと、Rhai エンジンがそれを実行して **中間表現 (IR)** 命令のシーケンスを構築します。これらの命令は、その後、オーディオサンプルごとに高性能な Rust エンジンによって実行されます。

---

## 2. API リファレンス

スクリプトは、オーディオロジックを構築するための制限された関数セットにアクセスできます。

### 入力と出力
- `input(idx)`: 指定されたインデックス `idx` の入力ポートに対する `Signal` ハンドルを返します。
- `output(idx, signal)`: 指定されたインデックス `idx` の出力ポートに `Signal` を接続します。

### 算術操作
- `add(sig1, sig2)`: 2 つのシグナルを加算します。新しい `Signal` を返します。
- `mul(sig, factor)`: シグナルに一定の係数を乗算します。新しい `Signal` を返します。

---

## 3. 例：カスタム・ゲイン

これは LimeStudio スクリプティングの「Hello World」です。

```javascript
// 1. ポート 0 から入力シグナルを取得
let x = input(0);

// 2. 0.5 倍（音量を半分）にする
let y = mul(x, 0.5);

// 3. 出力ポート 0 に送る
output(0, y);
```

### 内部で何が起きているか？
このスクリプトは以下の IR を生成します：
1. `LoadBuffer(Input_0)`
2. `MulConst(0.5)`
3. `StoreBuffer(Temp_1)`
4. `CopyBuffer(Temp_1, Output_0)`

---

## 4. なぜこれが安全なのか
- **無限ループの不在**: スクリプトはコンパイル時に一度だけ実行されます。もし実行に時間がかかりすぎる場合は、コンパイラがタイムアウトします。
- **リアルタイム違反（RT-Violations）の不在**: スクリプトはオーディオスレッドで実行されません。生成された IR のみがオーディオスレッドで実行されます。
- **決定論的**: 同じスクリプトは常に同じ IR を生成します。

---

## 5. 上級者向けのアドバイス
- **シグナルはバッファである**: `add` や `mul` を呼び出すたびに、一時的なバッファがコンパイラによって割り当てられます。
- **静的な生成のみ**: 現在、スクリプトは静的な IR 生成のみをサポートしています。スクリプトからの動的なパラメータ変調は、将来のアップデート（Level 3）で計画されています。
