# §SSS: AuthorityGlass Admission Policy

## 憲章: Blur は「意味」の敵である

LimeStudio における `AuthorityGlass` (Backdrop Blur) は、装飾（Prestige）のための機能ではない。それは**「情報の階層」と「因果の凍結」を物理的に強制するための特権的な手段**である。

Blur は輪郭を曖昧にし、情報の純度を低下させる。したがって、一般のウィジェットやパネルへの導入は、Trust UI の根幹を揺るがす「感染症」として厳格に排除される。

## 1. 認可される使用条件 (Admission Criteria)

`AuthorityGlass` の使用は、以下のいずれかの条件を満たし、かつ `Authority Layer` の管轄下にある場合に限定される。

### A. 因果関係の監査 (Forensic Inspection)
- **対象**: `Authority Drawer`, `Inspector Panel`
- **理由**: コードの真実（Real Reality）を閲覧する際、背後の「幻影（UI）」を視覚的に遮断し、ユーザーの意識をコンパイラの真実へ集中させる必要があるため。

### B. 現実の凍結 (Forensic Freeze)
- **対象**: `Forensic Freeze Panel`
- **理由**: バッファ上のデータをスナップショットとして固定し、時間軸から切り離された「証拠」であることを示すため。

### C. 致命的な矛盾の宣告 (Contradiction Alert)
- **対象**: `System Contradiction Modal`
- **理由**: UI の提示する状態とカーネルの真実が乖離した際、通常の操作を物理的に不能にし、システムの信頼性が損なわれたことを緊急告知するため。

## 2. 絶対禁止事項 (Hard Prohibitions)

以下のケースにおける `AuthorityGlass` の使用は、いかなる理由（「美しさ」「読みやすさ」等）があっても却下される。

- **一般設定画面**: 庶民の権力（Settings）に特権的な演出を与えてはならない。
- **ポップアップメニュー / ツールチップ**: 頻繁に現れる要素に Blur を適用すると、視覚的なノイズとなり、インターフェースの「確信」を奪う。
- **フローティングウィジェット**: 他のウィジェットの上に重なるだけの要素は、境界線を Blur ではなく Oklab のコントラストで定義すべきである。

## 3. 監査プロセス

すべての `AuthorityGlass` 使用箇所は、`Visible Compiler` の監査を受け、その妥当性が `Trust Ledger` に記録されなければならない。妥当性のない Blur は「不純物（Style Creep）」としてコンパイルエラーの対象となる。
