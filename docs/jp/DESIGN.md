# LimeStudio デザインドキュメント：可視化されたコンパイラ

## 1. 知覚 vs 現実：実行境界

LimeStudio は、「知覚（Perception - 人間の相互作用）」と「現実（Reality - マシンの実行）」を分離し、完全なリアルタイム安全性と決定論的な挙動を確保します。

### 1.1 知覚層 (Shell)
- **Crates**: `limestudio_vpl`, `limestudio_surface`
- **アイデンティティ**: GPU 最適化されたレンダリングのために `UiIndex` (u64) を使用。
- **責任**: インタラクション、レイアウト、視覚的な状態を管理します。オーディオグラフを直接変更することはなく、**意図 (Intent)** のみを発行します。

### 1.2 トランザクション層 (Bridge)
- **Crates**: `limestudio_core`
- **アイデンティティ**: `ViewCache` を介して `UiIndex` <-> `ULID` をマッピング。
- **責任**: UI の Intent をカーネルの Patch にコンパイルします。楽観的な UI 状態を管理し、それをカーネルのスナップショットと調和させます。「実行境界コントローラー」として機能します。

### 1.3 現実層 (Kernel)
- **Crates**: `dirtydata-runtime`, `dirtydata-core`
- **アイデンティティ**: グローバルに一意なエンティティ識別とコンテンツハッシュ（BLAKE3）のために `ULID` を使用。
- **責任**: 真実のソース。**JIT 実行プラン** を使用してオーディオグラフを実行します。完全な監査可能性のために状態の履歴（Lineage）を管理します。

## 2. JIT コンパイル：実行プラン

オーディオコールバックでのグラフ探索のオーバーヘッドを避けるため、LimeStudio はグラフのトポロジーを線形な **実行プラン (Execution Plan)** に変換します。

- **DspOp**: タイトなループで実行可能なプリミティブ命令セット（数学、状態、境界）。
- **PlanCompiler**: グラフをトポロジカルソートし、すべてのエッジに仮想レジスタを割り当てます。
- **スマートな最適化は行わない**: JIT は「Form Change（形態変化）」原則に厳格に従います。予測可能性を損なう可能性のある投機的な最適化や音響を考慮した最適化を試みることなく、グラフの構造をシーケンスに変換します。

## 3. Trust UI と敵対的検証

### 3.1 Trust UI (信頼の UI)
UI 要素は、カーネルの内部状態を公開するように設計されています。
- **モジュレーションリング**: パラメータのモジュレーション深度と現在の値を直接可視化します。
- **確信ビジュアライザー**: 観測された状態の `ConfidenceScore`（検証済み vs 推論済み）を表示します。
- **プロバナンストレース**: すべてのパラメータ値は、パッチ履歴内のその起源まで遡ることができます。

### 3.2 敵対的検証
カーネルは DSP の状態を常に検証します。
- **AssertRange**: すべてのノードの実行は安全チェックによって境界づけられています。
- **NaN/Inf アイソレーション**: 検出されたエラーは隔離され、フィードバックループによるスピーカーの破損を防止します。

## 5. デザインの強制：Lime の法

LimeStudio において、デザインは「提案」ではなく、**「コンパイルされた制約」** です。

### 5.1 Lime HIG v3.0 (法執行版)
**Human Interface Guidelines (HIG)** は、すべてのプラグインに対して義務的な視覚的および行動的ルールを定義します。
- **Matte & Solid**: 影、グロー、欺瞞的なライティングを禁止します。
- **形状による表現 (Shape Over Color)**: 幾何学的な状態変化を通じて、アクセシビリティと明快さを確保します。
- **8px グリッドの法**: アーキテクチャレベルでのスペーシングの一貫性を強制します。

### 5.2 LimeLint：信頼のコンパイラ
すべてのデザインは `LimeLint` によって監査され、ユーザーに嘘をついていないことが確認されます。
- **視覚監査**: 非準拠の UI 属性を拒否します。
- **フォレンジックな証言**: `cargo lime verify` を通じて、プリセットをその正確なソースロジックに結びつけます。
- **出荷の儀**: すべての提供されるプラグインが信頼できるものであることを保証するため、決定論的な「出荷の儀式 (`cargo lime release`)」を義務付けています。

## 4. SDK とエクスポーター

`dirtydata-sdk` は、すべての外部プラグインのための「法」を定義します。
- **plugin! マクロ**: ABI やホストの詳細を抽象化しつつ、開発者が純粋な DSP ロジックを書けるようにするカーネル DSL。
- **Transmuter (錬金器)**: ビジュアルグラフを、SDK コントラクトを使用するスタンドアロンのビルド可能な Rust プロジェクトに変換します。
